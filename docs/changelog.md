[2026-04-13] STREAMING: Added hybrid batch+streaming layer. Kafka (apache/kafka:latest, KRaft, port 9092) added to docker-compose.yml. Python producer (streaming/producer.py) replays raw.yellow_taxi_trips at configurable speed (default 60×) as JSON events on topic taxi-trips. PySpark Structured Streaming consumer (streaming/consumer.py) reads via foreachBatch and writes to raw.live_trips with deterministic event_id = concat(kafka_partition, '-', kafka_offset). UNIQUE(kafka_partition, kafka_offset) constraint enforces idempotency at DB level. Verified: 10,000 rows ingested, 12/12 dbt tests pass on stg_live_trips. Implementation notes: Python 3.14 (main venv) is incompatible with PySpark — uses .streaming-venv (Python 3.13). Java 23 removes Subject.getSubject() used by Hadoop 3.4.2 bundled in PySpark 4.1.1 — resolved by installing Java 21 LTS via Homebrew (openjdk@21) and setting JAVA_HOME in make streaming-consumer. Kafka connector + JDBC JARs pre-installed into PySpark's bundled jars dir (bypasses --jars/--packages flags which trigger the incompatible Hadoop FileSystem path resolution). (docker-compose.yml, scripts/streaming/migrate_live_trips.py, streaming/producer.py, streaming/consumer.py, streaming/download_jars.py, dbt/models/staging/stg_live_trips.sql, dbt/macros/clean_trip_fields.sql, dbt/models/staging/stg_yellow_taxi_trips.sql, dbt/models/staging/sources.yml, dbt/models/staging/schema.yml, requirements.txt, Makefile, .gitignore)
